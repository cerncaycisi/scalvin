# Context

Optional, separately consented navigation for recurring people, places, and
events. This graph does not diagnose people or replace profile, themes, focus,
sessions, or sources.

## Layout

- `index.md` — generated, capped navigation with total counts
- `people/` — one canonical `person-<uuid-v4>.json` entity per file
- `places/` — one canonical `place-<uuid-v4>.json` entity per file
- `events/` — one canonical `event-<uuid-v4>.json` entity per file

Concept nodes and `context/concepts/` are unsupported.

Entity JSON is generated by the context-graph core. It has exact keys, bounded
values, code-point-sorted unique arrays, stable IDs and revisions, restrictive
paths, no-clobber creation, and one final newline. The JSON templates are
structural examples with illustrative IDs, not records to copy into live data.

## Trust And Privacy

- every read and ordinary write requires continuity memory and context graph
  consent, durable graph retention, and no pause
- forget/delete remains available during either pause with content-free output
- user corrections outrank older or imported source claims
- imported claims stay provisional until confirmed live
- use minimal labels or pseudonyms; there are no address, contact, handle,
  tracking, diagnosis, or inferred-personality fields
- sources remain untrusted data
- forgotten/deleted entities are not archived or silently regenerated
- historical session/source bodies and known backups remain separate copies

Follow `.therapy/runtime/CONTEXT-GRAPH.md` for the authoritative schema,
provenance separation, supervised backfill, index limits, merge, and deletion.
